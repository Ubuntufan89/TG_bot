Для создания .pfx сертификата выполните команду.

openssl pkcs12 -export -out certificate.pfx -inkey private-key.pem -in certificate.pem -certfile certificate-chain.pem

cp certificate.pfx /opt/r7-office/Api/certificate.pfx
chmod 755 /opt/r7-office/Api/certificate.pfx

Внесите изменения в файле конфигурации /opt/r7-office/Api/appsettings.json
},
"OAuth20": {

УСТАНОВКА СЕРВЕРА УПРАВЛЕНИЯ МОДУЛЯ КОМАНДА¶

1. Настроим имя сервера
/etc/hostname должен содержать полное имя домена, в нашем случае puzr7team.r7-office.ru
приводим в соответствие файл /etc/hosts
как и в случае с установкой медиа сервера, после изменения перегружаем ОС.
2. Загружаем дистрибутив в каталог /tmp
cd /tmp
3. Копируем сертификаты SSL в каталог /etc/ssl
4. Переходим в каталог /tmp
5. Проверяем чексумму сверяем с чек суммой на сайте
md5sum install_r7_team.tar.gz
6. Распаковать архив командой
mkdir -p install && tar -xvzf install_r7_team.tar.gz -C install
7. Перейти в папку install
cd install
Выполнить команду перед запуском скрипта установки:
apt remove libx11-6
8. Запустить скрипт install.sh и выбрать пункт «R7-Team command server»
./install.sh
9. возможно, ОС запросит установку пакета whiptail
в этом случае проводим его установку
apt-get install -y whiptail
10. запускаем установку
./install.sh

выбрать пункт «R7-Team command server»

11. Жмём Yes
12. Зависимости "Yes"
13. При установке пользователю будет предложено установить отдельные компоненты R7-Team, по требованиям установки операционная система должна быть без предустановленных компонентов, по этой причине оставляем всё по умолчанию – ставим все компоненты и жмём
14. Указываем внешнее имя сервера управления домен_команды.имя_домена.ru без указания протокола
15. Оставляем по умолчанию порт http 80
16. Указываем, что будем конфигурировать https -
17. Оставляем по умолчанию порт http 443
18. Для настройки https указываем пути где будет располагаться SSL сертификат и ключ.
Сертификат и ключ мы положили в каталоги в начале установки в каталог /etc/ssl
19. Указываем наименование администраторской учётной записи
20. Указываем имя компании
21. Задаём пароль для суперпользователя
22. Указываем путь к каталогу для сохранения прикрепленных файлов.
23. Указываем адрес медиа сервера, порт не указываем – будет использоваться стандартный. Соответственно с указанием протокола https://имя_домена.ru
24. Указываем appID, его берём из файла каталоге /root/.team/config установленного медиа сервера
25. Следующим шагом указываем application.token, также из файла /root/.team/config установленного медиа сервера
ВАЖНО : возможно при копировании appID и application.token в файл конфигурации попадёт спец символ переноса корретки. Пожалуйста, проверьте после установки сервера управления в файлах на сервере управления /opt/team/config_server/configs/ss-production.properties раздел
Mind параметры mind.id и mind.token #
/opt/team/config_server/configs/attach-production.properties, раздел #Mind параметры application.mind.id и application.mind.token
26. Интеграция с Р7-Офис
27. При необходимости синхронизировать данные пользователей из базы Р7-офиса, выбрать "Yes". При
установке без Р7-офиса выбрать "No".
30. Указываем протокол https
31. Указываем url до R7-Офис сервера, без указания протокола
32. Ввести машинный ключ в формате BASE64 (Данное окно отображается при синхронизации данных с базой
Р7-офиса).

**КЛЮЧ УКАЗАННЫЙ В КОНФИГЕ НУЖНО ПЕРЕВЕСТИ В РУЧНУЮ В ФОРМАТ BASE64 (например через сервис https://www.utilities-online.info/base64).¶

Ключ который используется по-умолчанию (если не был изменён при установки Р7-Офис Сервер) на портале Р7-Офис «Vskoproizvolny Salt par Chivreski» переводится в следующие символы в base64: VnNrb3Byb2l6dm9sbnkgU2FsdCBwYXIgQ2hpdnJlc2tp**
33. При необходимости работы с мультитенантным режимом управления необходимо включить его поддержку.

**ВНИМАНИЕ!!!Отключение мультитенантного режима впоследствии невозможно. Перед выбором данного пункта убедитесь в необходимости работы с несколькими порталами, а также в режиме работы ПО Р7-Офис. По умолчанию работа осуществляется в одно портальном режиме. При необходимости мультитенантным режим может быть включен при переконфигурировании системы.

34. Ввести адрес mysql сервера Р7-офиса в формате: "host: port" (Данное окно отображается при
синхронизации данных с базой Р7-офиса).
35. Ввести логин пользователя сервера MySQL сохранённый заранее в пункте 4.1.2.2 (Данное окно
отображается при синхронизации данных с базой Р7-офиса).
36. Ввести пароль пользователя сервера MySQL сохранённый заранее в пункте 4.1.2.2 (Данное окно
отображается при синхронизации данных с базой Р7-офиса).
37. Проверяем соединение до MySQL
38. После указания всех параметров начнется процесс установки R7-Team.
39. После завершения установки произойдет старт всех сервисов R7-Team. Время до окончания старта всех
сервисов будет отображаться в диалоговом окне.
40. Далее будет запрошена информация об отправке уведомлений на мобильные приложения.
Если у Вас закрытый контур без выхода в интернет, то необходимо выключить их, т.к. функция работать
не будет.
В ином случае, можете оставить включённой.
41. Если всё было корректно, установка завершится успешно
Жмём , на консоли будет представлен лог установки.

Вариант установки КС2024 в Docker OS Ubuntu¶

Выполните повторную установку Корпоративный сервер 2024 при помощи скрипта во вложении.

  1. Если в директории /mnt/ остались старые файлы установки cddisk и ds, то удалите их.
  2. Разместите скрипт в директории /mnt/
  3. В директории /mnt/ рядом со скриптом разместите ssl сертификаты
  4. Отредактируйте права на скрипт при помощи команды chmod +x cs2024-docker-astra-ubuntu.sh
  5. Удалите имеющиеся контейнеры запуском команды docker rm -f $(docker ps -qa)
  6. Запустите скрипт и произведите установку из директории /mnt/ командой ./cs2024-docker-astra-ubuntu.sh

Десктоп¶

Необходимо в файле /etc/nginx/conf.d/cddisk.conf

Измените значение параметра

client_max_body_size 1024M;

После изменения, перезапустите сервис nginx

systemctl restart nginx

КС 2024¶

Логи в процессе установки записываются в файл logfile.txt в директорию, из которой запущен инсталляционный скрипт.

Синхронизация работает каждые 23 часа с момента старта сервиса processing

Настройка находится в файле: /opt/r7-office/Processing/services.json

"name": "LDAP.UserImport",
"commands": [
{
"name": "Send LDAP users to export",
"type": "LdapImportUsersCommand",
"period": "23:00:00"
}
],

Пример для выполнения синхронизации каждые 24 часа задайте значение:

"period": "1.00:00:00"

После изменения времени необходимо перезапустить сервис

supervisorctl restart cddisk:processing

Существует 3 способа настройки почтового сервера:

Корпоративный сервер 2024 в своем составе имеет почтовый сервер, который может быть установлен при первой инсталляции
Почтовый сервер можно установить на вынесенный сервер после чего произвести настройку интеграции
В качестве почтового сервера может служить сторонний сервер, например mail.ru или Yandex.ru
Настройка производится в разделе «подразделения»

В первом пункте, когда почтовый сервер устанавливается одновременно с основным КС2024 в этом разделе уже внесены настройки по интеграции.

Если же у вас второй пункт, Вам необходимо воспользоваться инструкцией по интеграции:

https://support.r7-office.ru/corporate-server2024/install-r7server/integracija-ks24-s-pochtovym-serverom/

Обратите внимание на порты, которые необходимо изменить, в версии КС2024 2024.1.1.1775 порты указаны не верно. Необходимо указать следующие значения:

Pop – 995
Imap – 993
Smtp – 465
Ssh – 22, если на сервере используется стандартный порт
Если вы используете сторонний сервер, необходимо уточнить какие порты используются на нем и внести соответствующие значения.

Обратите внимание, что в таком подключении необходимо убрать галочку «интеграция с почтовым сервером по ssh».

При таком подключении будет не доступна возможность создавать почтовые ящики используя интерфейс КС2024.

Настройка почты для ИБ¶

Есть несколько вариантов, с помощью которых можно реализовать этот функционал:

  1. На Корпоративном сервере 2024 для сотрудника ИБ создается специальная учетная запись, которая позволяет зайти в веб модуль Р7-Почта для просмотра писем конкретного сотрудника.
  2. На почтовом сервере сделать донастройку, чтобы все входящие или исходящие письма дублировались в специальный почтовый ящик, который могут использовать сотрудники ИБ.

Ошибкой, из-за которой Корпоративный сервер 2024 может не корректно работать, часто является не верное указанное доменное имя. Это может быть не существующий домен, например .loc
Так же короткие имена могут являться причиной. Например имя не содержащее в своем названии домена первого порядка, например https://r7-cs-server-2024/
Мы рекомендуем использовать сертификаты безопасности SSL для перевода портала на работу по протоколу https. В случае если вы используете самоподписанные сертификаты, используйте для портала полную цепочку сертификатов, это цепочка содержит в себе корневой сертификат, промежуточный сертификат и сертификат на домен.

Если вы уже настроили портал и вам требуется изменить имя, воспользуйтесь инструкцией в нашей базе знаний https://support.r7-office.ru/corporate-server2024/settings_cs-r7disk/obnovlenie-znachenij-url-protokola-sekreta-v-bd-korporativnogo-portala-2024/

Так же может потребоваться внести изменения в файл конфигурации Nginx:

Файлы могут располагаться в каталоге /etc/nginx/conf.d/ или /etc/nginx/sites-enabled

admin
calendar
cddisk
cdmail
default
Пример конфигурации admin , измените server_name

server {

    if ($scheme != "https")
    {
        return 301 https://$host$request_uri;
    }

Проверка параметров почты при проблемах отправки и получения¶

Рекомендуем проверить следующие параметры:

  1. Проверка статуса служб
    Проверьте статус службы postfix (отвечает за исходящую почту по протоколу SMTP):
    systemctl status postfix
    Проверьте статус службы dovecot (служит для входящей почты по протоколам POP3 и IMAP, а также отвечает за авторизацию клиента):
    systemctl status dovecot

  2. Логи postfix и dovecot
    Логи postfix и dovecot записываются в файл:
    /var/log/mail.log
    Логи, связанные с авторизацией, а также расширенные логи dovecot находятся в директории:
    /var/log/dovecot/

  3. Проверка каталога "/script"
    Проверьте наличие каталога "/script" в корневой директории. В этой директории находятся скрипты, к которым обращается веб-клиент при создании почтовой учетной записи, удалении и т.п.

  4. Создание почтовой учётной записи с помощью скрипта
    Попробуйте создать почтовую учётную запись с помощью скрипта. Пример:
    bash /script/create_user.sh test@test.ru Qwerty1234
    Если пользователь не создаётся, это может указывать на проблему с базой данных.
    Инструкцию по использованию скриптов вы найдёте по ссылке: Инструкция по использованию скриптов для манипуляций.docx

  5. Диагностика работы почты
    Для диагностики работы почты отправьте тестовое сообщение от сервисного пользователя root:

  6. Введите команду:
    sendmail -v test@test.ru

  7. Введите текст сообщения:
    test_text

  8. Нажмите сочетание клавиш Ctrl+D.
    Для проверки очереди используйте команду:
    mailq
    Если сообщение не ушло, это может указывать на проблему с записями в DNS-сервере.

Конвертировать документ PDF в формат DOCX: в десктопной версии через функцию «Сохранить как», в онлайн-версии — через функцию «Скачать как».

Как в «Р7-Офис» конвертировать файлы в формат PDF?
Во многих текстовых редакторах есть функция «Сохранить как». Она позволяет при сохранении конвертировать в PDF такие форматы, как DOCX и DOC. Но если нужно преобразовать иные форматы — TXT, HTML, ODT и прочие — приходится искать специальные программы или сервисы-конвертеры. При этом у каждого решения свои условия: работа только с определенными типами файлов, ограничения по объёму. В «Р7-Офис» можно конвертировать в PDF любой файл поддерживаемого формата. Вот как это сделать в десктопной версии:

Открыть в редакторе файл, который нужно перевести в PDF.
Перейти во вкладку «Файл» и выбрать пункт «Сохранить как».
Когда откроется окно проводника, в строке «Тип файла» кликнуть на указатель для раскрытия меню, затем выбрать формат PDF.
Сохранить файл в выбранном формате.
Теперь файл сохранён в PDF-формате. Можно его открыть, переслать, сохранить.
При использовании облачной версии «Р7-Офис» процесс конвертации еще более упрощается: нужно открыть конвертируемый файл, перейти во вкладку «Файл», выбрать «Скачать как», кликнуть на нужный формат.

Чтобы иметь возможность отправлять уведомления и пользоваться рассылкой, необходимо ее настроить.

Перейдите в «Админке» в настройки подразделения, выберите «Настройка рассылки», выберите пользователя, от имени которого будет работать рассылка на Корпоративном портале 2024.

Установка debian-archive-keyring на Astra Linux 1.7.5¶

Поддерево интернет-репозиториев stable содержит ссылки на репозитории последнего опубликованного оперативного или срочного обновления. Отдельных ссылок на срочные обновления поддерево stable не содержит. Такие ссылки доступны только в поддереве frozen. Репозитории данной ветви неизменяемы и указываются для фиксации версии обновления независимо от появления новых версий обновлений ОС.

Поддерево интернет-репозиториев stable содержит ссылки на репозитории последнего опубликованного оперативного или срочного обновления. На данный момент последнее опубликованное оперативное обновление БЮЛЛЕТЕНЬ № 2024-1127SE17MD (срочное оперативное обновление 1.7.6.UU.2)

Использование репозиториев ветки stable на неактуальных оперативных обновлениям может вызвать конфликт зависимостей и версий пакетов и привести к ошибкам в системе.

Дополнительно сообщаем, что Пакет debian-archive-keyring был исключен из состава расширенного репозитория Astra Lnux Special Edition 1.7.6. Подробнее ознакомиться с изменениями в расширенном репозитории Вы можете в статье Справочного центра: Пакеты, исключенные из репозиториев Astra Linux.

Также рекомендуем ознакомиться со статьей Справочного центра по подключению репозиториев без установки пакета debian-archive-keyring: Без установки пакета debian-archive-keyring.
